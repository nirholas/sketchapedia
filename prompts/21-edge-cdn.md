# 21 — Edge CDN Worker

## Project context

Keyframes and transition clips should be served from the edge, not the origin cache server, to achieve the < 50ms steady-state latency Sketchapedia's economics require. The edge worker sits in front of R2, serves signed-URL artifacts, applies tenant-scoped caching, strips PII, and handles negotiation of client codec capabilities (AV1 vs. H264). See `prompts/00-vision.md`.

## Your task

Implement `packages/edge-worker/` — a **Cloudflare Workers** (with optional Deno Deploy / Vercel Edge adapter) service. It is stateless, fast, and lean (bundle budget: < 100 KB compressed).

## Technical requirements

- Runtime: **Cloudflare Workers** (`@cloudflare/workers-types` v4), deployed via **Wrangler**.
- Bindings:
  - `R2_ARTIFACTS` — R2 bucket holding keyframes/transitions/hitmaps.
  - `KV_SIGNED_KEYS` — rotation of signing keys.
  - `KV_TENANT_CONFIG` — per-tenant caching + codec preferences.
  - `QUEUE_ANALYTICS` — async telemetry.
- **Signed URLs**: HMAC-SHA256 over `(artifactId, expiresAt, tenantId)` with a rolling key from KV. Verified on every request.
- **Caching**: edge cache API (`cache.put(request, response)`) with `Cache-Control: public, max-age=31536000, immutable` since artifacts are content-addressed and never change.
- **Codec negotiation**: `Accept` header parsed; if AV1 supported serve AV1 clip, else H264. Two artifact IDs (AV1 + H264) pre-generated by prompt 18; this worker picks.
- **Range requests**: supported natively via R2 + worker passthrough for video streaming.
- **Hot-path purge**: origin can call `POST /purge { artifactId }` with an admin secret to invalidate edge cache (rare — content-addressed means mostly we just roll keys).

## Routes

- `GET /sk-artifacts/:signedKey/:artifactId.:ext` — resolve, verify signature, check edge cache, fall back to R2, return bytes.
- `POST /purge` (admin) — invalidate a single artifact across all edges.
- `GET /codec-hint` — returns the preferred codec for the client based on UA and Accept.
- `GET /healthz` — 200 OK.

## Signing contract

```
signedKey = base64url( hmac_sha256( tenantId + ':' + artifactId + ':' + expiresAtMs, rollingKey ) )[0..31]
url = https://edge.sketchapedia.com/sk-artifacts/<signedKey>/<artifactId>.<ext>?e=<expiresAtMs>&t=<tenantId>
```

Keys rotate weekly; workers accept signatures from the last 2 keys for seamless rollover.

## Implementation mandates

- Cold boot < 10ms; reuse HMAC key cache across requests.
- Non-blocking telemetry: `waitUntil(queue.send(event))` — don't delay the response.
- Fail closed: missing signature, expired, or tenant mismatch → 403.
- Never log artifact bytes; only `artifactId`, `tenantId`, HTTP status, latency.
- CORS configured for scene artifacts: `Access-Control-Allow-Origin: <configured origin>` per tenant from `KV_TENANT_CONFIG`.
- Handle `If-None-Match` / `ETag` correctly.
- Tight bundle: tree-shake aggressively; no `@aws-sdk/client-s3` (use R2 binding directly).

## Test plan

- `miniflare` for local workers testing.
- Scenarios:
  - Valid signed URL → 200 + content from R2.
  - Expired signature → 403.
  - Wrong tenant → 403.
  - Key rotation: old-key URL still works during the grace window; fails after.
  - Range request: `Range: bytes=0-1024` returns 206 with partial content.
  - Codec negotiation: AV1-capable UA gets AV1; Safari < 17 gets H264.
- End-to-end from real Cloudflare deployment (CI job against a test zone) hitting R2 origin, measuring p99 latency.

## Deliverables

- `packages/edge-worker/src/{worker.ts, sign.ts, cache.ts, negotiate.ts, config.ts}`.
- `packages/edge-worker/wrangler.toml` with env bindings.
- Deno/Vercel adapters behind build conditionals (`packages/edge-worker/src/adapters/`).
- `packages/edge-worker/README.md` with deployment runbook.

## Acceptance criteria

- p99 edge latency < 50ms for warmed objects; < 200ms for cold.
- Bundle size < 100 KB compressed.
- All scenarios green.
- Secret rotation tested end-to-end.

## Non-goals

- No generation logic.
- No authentication beyond signed URL verification.
- No per-user analytics — only tenant-level aggregates.
