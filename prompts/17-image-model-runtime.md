# 17 — Image Model Runtime

## Project context

The image model produces each scene's keyframe. For Sketchapedia to feel cohesive across intents, the model must obey the LLM's layout (so the hitmap aligns to pixels) and preserve style from the previous keyframe (so transitions don't jump). Real model. Real GPU. No placeholder. See `prompts/00-vision.md`.

## Your task

Implement `packages/model-image/` — a service that runs **FLUX.1-dev** (primary) with **IP-Adapter** for style-reference conditioning and **ControlNet (region/layout)** for coordinate alignment. Expose a Python inference server (behind the orchestrator) plus a TypeScript client. SDXL adapter as a secondary option.

## Technical requirements

- Inference runtime: **Python 3.11**, **PyTorch 2.4+**, **diffusers ≥ 0.30**, **xformers** / **flash-attn** where supported, **bitsandbytes** for 8-bit where VRAM-constrained.
- Server framework: **FastAPI** with async endpoints; Uvicorn workers.
- **FLUX.1-dev** loaded via `diffusers.FluxPipeline`. Use `bfloat16` and `enable_model_cpu_offload()` on 24GB GPUs; `enable_sequential_cpu_offload()` for 16GB fallback.
- **IP-Adapter** (`ip-adapter-plus`) for style-reference conditioning — previous keyframe fed as reference to preserve palette, rendering mode, line weight.
- **ControlNet for regions**: use a regional-prompt pipeline (or `ControlNetUnion` if available) that accepts a mask + per-region prompt pairs derived from the `LayoutPlan.hitmapDraft`. Enforces spatial layout so hitmap coords land near pixel regions.
- **LoRA**: support tenant-specific style LoRAs loaded at request time (pre-warmed in memory; LRU cache of last 16 LoRAs).
- **Scheduler**: `FlowMatchEulerDiscreteScheduler` (FLUX default); step count configurable (default 30 for quality, 20 for speed).
- **Output**: WebP lossless or AVIF encoded by `pillow-avif-plugin`; 1920×1080 default; adaptive resolution honored per request.
- **GPU acceleration primitives**: `torch.compile` with `reduce-overhead` mode; `SDPA` attention via `torch.nn.functional.scaled_dot_product_attention`.

## HTTP API (internal, behind orchestrator)

```
POST /generate
{
  "prompt": "...",
  "style_reference_url": "https://...",   // signed URL to previous keyframe
  "regions": [ { "bbox": [x,y,w,h], "prompt": "...", "role": "..." } ],
  "render_mode": "illustration" | "photoreal" | ...,
  "size": { "width": 1920, "height": 1080 },
  "seed": 12345,
  "steps": 30,
  "guidance_scale": 3.5,
  "lora": [ { "name": "tenant-style-v3", "weight": 0.7 } ]
}
→ 200 { "image_url": "s3://...", "hash": "...", "width": ..., "height": ..., "latency_ms": ..., "seed": ... }
```

Streams progress via Server-Sent Events (percent + eta).

## TypeScript client (the orchestrator consumes this)

```ts
interface ImageModelClient {
  generate(input: ImageGenInput, signal?: AbortSignal): AsyncIterable<ImageGenEvent>;
}
```

Handles SSE decoding, retries on 5xx (idempotent with same seed), and uploads the generated bytes to the cache-server (prompt 20).

## Implementation mandates

- Warm-start: the pipeline stays loaded across requests. Cold boot (initial model load) ~60s; GPU dispatcher (prompt 22) maintains a warm pool.
- **NSFW filter** (safety checker) always on in production; overrideable only for tenants on an allowlist.
- **Seed** determinism: same input + seed produces byte-identical output (test proves it on a reference fixture).
- **Memory accounting**: log VRAM high-water mark per request; shed workload when >90%.
- **Telemetry**: OTel spans across `prepare`, `inference`, `encode`, `upload`.
- **Error classification**: `OOM`, `TIMEOUT`, `CONTENT_FILTERED`, `INVALID_CONTROLNET`, `BAD_REFERENCE`.
- Model binary + LoRAs stored on a fast local SSD with a shared-read cache; hot-swap without full server restart.

## Regional-prompt alignment strategy

The LLM (prompt 16) emits `hitmapDraft` with regions + semantic labels. This service:

1. Converts each region to a binary mask at keyframe size.
2. For each region, derives a per-region prompt like `"a blue reservation button with the text 'Reserve'"` from the hitmap item's `ariaLabel` + `role`.
3. Passes the mask + prompts through regional prompting (either via `regional_diffusion` pipeline or by injecting attention masks during sampling).
4. Produces an image whose semantic regions land near the LLM-declared coordinates. The vision model (prompt 19) then grounds and corrects residual drift.

## Test plan

- Real GPU in CI (A10G or L40S via self-hosted runner, gated to main branch).
- Golden seed reproducibility: same inputs + seed → same SHA256 of output bytes.
- Style continuity: generate 10 scene pairs with IP-Adapter enabled; human review assigns continuity score ≥ 8/10 on average.
- Region alignment: for 20 fixtures, run vision (prompt 19) and compute IoU of detected vs. declared regions; mean IoU ≥ 0.7.
- Benchmark: 1920×1080 at 30 steps on A10G ≤ 7s; L40S ≤ 3s.
- NSFW gate: red-team prompts all blocked.
- Memory: 100 consecutive requests without OOM.
