# 28 — Documentation Site

## Project context

Sketchapedia's promise is radical enough that the docs are the sales pitch. The docs site must explain the philosophy, walk through building a first app, document every public API, render interactive demos, and ideally render parts of itself using Sketchapedia (the ultimate dogfood). See `prompts/00-vision.md`.

## Your task

Implement `apps/docs/` — the public documentation site at a dedicated URL (e.g. `docs.sketchapedia.com`). Built with **Astro 4+** + **Starlight** for the reference sections; interactive demos embedded via the `@sketchapedia/client-react` package. Extensive MDX content.

## Technical requirements

- **Astro + Starlight** for content and navigation.
- **MDX** pages so demos can be embedded via React components.
- Deployed as a static site to Cloudflare Pages; interactive demos connect to a dedicated demo backend tenant.
- Full-text search via **Pagefind** (Starlight default).
- Dark + light themes; honors `prefers-color-scheme`.
- i18n scaffolding for English (baseline) + one additional locale as a proof (e.g. Japanese) to validate the LLM locale path.

## Content structure

- `/start` — 5-minute quickstart.
- `/concepts` — MaaR, hybrid architecture, hitmap alignment, cache-first economics.
- `/guide/react` — building a Sketchapedia React app end-to-end.
- `/guide/scenes` — authoring scenes, layout plans, style references.
- `/guide/accessibility` — the a11y story; why the DOM overlay matters.
- `/guide/ops` — self-hosting, scaling, observability.
- `/examples` — live reproductions of the 4 reference apps (prompts 29–32) embedded inline.
- `/reference/*` — autogenerated from `TSDoc` across packages: protocol types, client-core, client-react, server-gateway, orchestrator, models, cache, edge, CLI. Uses **Typedoc** + **typedoc-plugin-markdown** to emit MDX.
- `/threat-model` — renders `SECURITY.md`.
- `/blog` — optional; leave scaffold.

## Interactive demos

- Each example page embeds a `<Sketchapedia>` component pointing at a demo tenant.
- Source-of-truth code snippets syntax-highlighted via **Shiki**.
- Demos gracefully degrade when the demo backend is unreachable (static screenshot fallback).

## Implementation mandates

- Fully typed MDX front matter.
- Builds in < 3 minutes.
- Passes Lighthouse 95+ on performance, accessibility, best practices, SEO.
- Every code snippet tested by extraction → compile check (`@astrojs/check` + a custom extractor script).
- Generated reference pages kept in sync automatically — a CI job fails the PR if docs drift from TSDoc.
- Deploy preview on every PR via Cloudflare Pages integration.

## Test plan

- `pnpm --filter @sketchapedia/docs build` succeeds with zero broken links (`astro-link-checker`).
- Lighthouse CI enforces thresholds.
- Playwright smoke on deployed preview — loads the homepage, follows the quickstart link, interacts with an embedded demo.
- Axe-core on every page.

## Deliverables

- `apps/docs/` — Astro project.
- `apps/docs/scripts/extract-reference.ts` — pulls TSDoc → MDX.
- CI workflow `.github/workflows/docs.yml`.
- `apps/docs/README.md`.

## Acceptance criteria

- Public URL live with content + interactive demos.
- Lighthouse thresholds met.
- Reference autogen green.

## Non-goals

- No CMS.
- No user accounts or gated content.
